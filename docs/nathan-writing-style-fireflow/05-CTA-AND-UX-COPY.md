# CTA and UX Copy

## Core rule

Every CTA should name the action and match what happens next.

## Preferred CTA style

Use a direct verb and a clear destination or result.

Good:

- Enter FireFlow
- Explore with Nathan
- Turn on Operator Notes
- Open the product dossier
- Compare selected products
- Start a consumer case
- Start a vendor case
- Review the order exception
- Continue to billing
- View Nathan's experience
- Open Nathan's résumé
- Discuss the role

Weak:

- Learn more
- Get started
- Discover
- Explore now
- Unlock insights
- Click here
- See magic

## Confirmation copy

A confirmation should state what happened and what the visitor can do next.

Good:

> Case FF-2047 was added to the synthetic queue. Review the routing decision or follow the case through resolution.

Weak:

> Success! Your experience has been elevated.

## Error copy

State the problem, preserve the user's work when possible, and provide a recovery action.

Good:

> The case could not be added to the local queue. Your selections are still here. Try again or copy the case summary.

## Modal copy

- Name the purpose in the title.
- Keep the first paragraph short.
- Show what information will be used.
- Make cancel and close behavior clear.
- Do not pretend a local demonstration was submitted to a real company.

## Labels

Prefer precise labels such as Product format, Issue type, Evidence collected, Case owner, Customer update, and Next action.
