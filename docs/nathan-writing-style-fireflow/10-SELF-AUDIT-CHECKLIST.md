# Self-Audit Checklist

Run this check over every visible string before finishing.

## Voice

- Does the writing sound human, calm, and specific?
- Does it sound like an operator rather than a brochure?
- Is the confidence based on evidence?
- Did the copy preserve Nathan's directness without copying drafting errors?

## Clarity

- Is the point clear in the first sentence?
- Does each paragraph have one main purpose?
- Is the actor or owner named when it matters?
- Is the next action obvious?

## Claims

- Is every experience claim true and interview-defensible?
- Are synthetic values labeled?
- Are public facts separated from editorial interpretation?
- Does SAP language avoid implying direct tenure or implementation experience?

## UX copy

- Does every CTA describe the action?
- Does the label match the destination?
- Are success and error states specific?
- Does a modal explain what happened without pretending a real submission occurred?

## Style

- No em dash sentence separators
- No decorative arrows
- No unnecessary hyphens
- No generic AI phrases
- No repeated self-praise
- No vague Learn more CTA when a specific action is possible
- No underline-only hover behavior that weakens the visual system

## Final read aloud

Read the copy aloud. Rewrite anything that sounds like a corporate script, legal disclaimer, or AI landing page.
