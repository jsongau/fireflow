# Nathan Writing Style Pack

This package gives Claude a reusable writing system for Nathan J. Song's projects, especially FireFlow and employer-facing portfolio work.

It is designed to prevent three common failures:

1. Generic corporate language that hides the real operational idea.
2. Overstated claims that become difficult to defend in an interview.
3. Decorative UX copy that sounds impressive but does not help the visitor understand what to do or why it matters.

## How to use it

Upload this ZIP to the Claude project or place the folder in the repository. Tell Claude to read `SKILL.md` and every numbered file before editing visible copy.

For FireFlow, also provide the repository and the target job description. The style pack controls the voice. The repository and job description control the facts.

## Important distinction

Nathan's final voice is not a transcript of his fast drafting style. It keeps his directness, curiosity, urgency, and practical reasoning while correcting grammar and removing repetition.
